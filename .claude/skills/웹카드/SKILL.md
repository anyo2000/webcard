# /웹카드 — 제작 절차 (순서 강제)

> **왜 스킬인가**: 단계 누락 사고 방지 (2026-07-24 OG 썸네일 빼먹고 발행 → 카톡 빈 링크 사고).
> 디자인·문구·폰트·컴플라이언스 등 **내용 규칙은 전부 레포 CLAUDE.md가 single source** —
> 이 스킬은 순서와 게이트만 강제한다. 규칙이 바뀌면 CLAUDE.md만 고친다.

## 0. 규칙 로드
- 레포 `CLAUDE.md`의 **제작 규칙 · 문구 지침(1~11) · 원문 충실도 HARD RULE** 준수

## 1. 원문 게이트 — 없으면 착수 금지
- mybrain `knowledge/monthly/{YYMM}/본사/`에서 원문 검색
  - ⚠️ 검색 결과를 `head`로 자르다 원문을 놓친 사고 있음 (2026-07-24 통합암) — 전체 확인
- 원문 없으면 차장님께 공문 캡처/텍스트 요청. **원문 없는 숫자·조건은 쓰지 않는다**
- 차장님 구두 정보는 출처를 분리 표기 (공문 vs 안내)
- 담보 약칭은 `mybrain/knowledge/base/담보명_약칭사전.md` 확인

## 2. 구상 합의 — 빌드 전 게이트
- 페이지별 구성 제안: 기본 4~6장 (간단 요청이면 4장 — 표지 / 왜·무엇 / 행동 / 체크포인트)
- 표지 색 제안: 시리즈 형태 유지 + **카드마다 다른 팔레트** (블루라벤더·민트틸·앰버피치 사용됨)
- 언노운 질문 1~3개(톤·범위·해석 문구·시리즈 연결)를 함께 제시 → **승인 후 빌드**

## 3. 빌드
- 최신 카드의 index.html을 베이스로 복제 후 수정 (구조·모션·시트·툴팁 재사용)
- 폴더 `{YYMM}/{영문슬러그}/` — 한글 경로 금지
- 푸터 관리번호 = 기존 최대 +1 (`grep -h "영업컨설팅파트" 26*/*/index.html`로 확인)
- `analytics/track.js` 삽입 + 탭 핸들러에 `wc('tap', …)`

## 4. 검증
- 로컬 서버(8899) + `_shot.html`로 **전 슬라이드 스크린샷** → 넘침·겹침·어색한 줄바꿈 확인 후 수정
- **공문과 1:1 대조** (숫자·조건·담보명) + FP 행동 누락 확인

## 5. OG 썸네일 — 누락 금지
- CLAUDE.md "카톡 썸네일(OG)" 절차: 표지 팔레트로 800×418 컴포지션 → `og.png`(1600×836) → 메타 삽입
- og:title = 최단 팩트형, og:description = 핵심 한 문장 (문구 지침 9번)

## 6. 총괄장
- 루트 `index.html`에 카드 추가 (긴급·최신이 맨 위), 제목 = 최단 팩트형 + 상품군 접두어(SI 등)

## 7. 완료 보고
- 자동 미리보기(`open http://localhost:8899/{경로}/`) — CLAUDE.md "완성 후 자동 미리보기" 규칙
- 스크린샷 근거로 구성·원문 대조 결과 보고
- **푸시는 차장님 지시 시** `/push` 스킬로. 이미 공유된 링크의 OG를 바꿨으면 `?v=N` 재공유 안내
