# webcard — 인터랙티브 교육자료 (웹카드)

## 이 레포는 뭐하는 곳?
카드뉴스(이미지)를 대체하는 **인터랙티브 HTML 교육자료**의 실행물 저장소.
스마트폰에서 세로로 밀어 넘기고, 항목을 눌러 상세를 여는 방식.

**카드뉴스와의 차이 = 링크가 계속 산다.**
- 카드뉴스: 카톡방에 묻히면 끝 (3주 뒤 못 찾음)
- 웹카드: 링크가 남고, 내용 수정 시 **이미 뿌린 링크에 즉시 반영** (재배포 불필요)

## 역할 분담 (중요)
- **지식·원천 데이터** (인수기준 원문, 담보명 약칭사전, 스타일가이드) → `mybrain`
- **실행물(HTML)** → 여기
- 카드뉴스·영상·교안과 동일 패턴. HTML은 **소스 = 배포물**이므로 소스를 따로 보관하지 않는다.

### 다른 레포와 헷갈리지 말 것
| 레포 | 무엇 |
|------|------|
| `webcard` (여기) | **정규 공지·교육자료**. 카드뉴스 후계. 링크가 오래 산다 |
| `new-edu` | 정규 **밖**의 참여형/실험 앱 (챔피언십·실시간 투표·행사용). 한 번 쓰고 끝 |
| `cardnews` | 이미지 카드뉴스 (기존 방식) |
| `mybrain` | 지식 원천 (private). 산출물 두지 않음 |

## 배포
- **PUBLIC 레포 + GitHub Pages** (private은 무료 Pages 불가 → 반드시 public)
- 배포 주소: `https://anyo2000.github.io/webcard/{YYMM}/{슬러그}/`
- 목록 페이지: `https://anyo2000.github.io/webcard/` ← FP에게 이거 하나만 알려주면 지난 자료도 다 찾음
- `robots.txt`로 검색엔진 차단 (링크 아는 사람만 접근). 단 **GitHub 저장소 자체는 공개**임을 항상 인지할 것

## 폴더 규칙
자료마다 레포를 새로 만들지 않는다. **`{YYMM}/{영문슬러그}/index.html`** 로 누적.

```
webcard/
  index.html            # 목록 페이지 — 새 자료 추가할 때마다 카드 1개 append
  2607/
    si-uw/index.html    # SI 경증질환 심사기준 변경 (7.20 시행)
  _template/            # 다음 달 재사용 뼈대
  robots.txt
```

- **폴더명은 반드시 영문 슬러그.** 한글 경로는 카톡에서 `%EA%B2%BD...`로 깨져 링크가 흉해진다.
- 새 자료 추가 시 `index.html` 목록에 카드 1개 추가하는 것을 잊지 말 것.
- **사용측정**: 새 카드 `</body>` 직전에 `<script src="../../analytics/track.js"></script>` 삽입 + 탭 핸들러에 `if(window.wc)wc('tap','키')` 추가. 방문·슬라이드깊이·체류는 자동. 상세: `analytics/README.md` (수집처=회사 구글계정 Apps Script→시트, 대시보드는 웹앱 URL).

## 제작 규칙 (2607 si-uw에서 확립)

### 구조
- 세로 풀스크린 스냅 스크롤 (카드뉴스와 동일한 동선 — 밀어서 넘김)
- 상단 인스타식 진행바
- 항목 탭 → 하단 시트(bottom sheet)로 상세
- 용어에 점선 밑줄 → 탭하면 툴팁 (진짜 모르는 용어만. 자명한 건 달지 말 것)

### 폰트 크기 — HARD RULE
**본문 18px / 캡션 15px가 바닥. 그 아래로 내려가지 않는다.** (FP 상당수가 50대 이상)
- 유일한 예외: 컴플라이언스 문구 (9.5~10px, **의도적으로** 작게)

### 컴플라이언스 (필수)
- 상단 고정: `본 자료는 교육용으로 온라인 매체(SNS등) 및 고객 설명, 광고자료로 활용 행위는 법으로 금지됩니다.`
- 하단 고정: 좌 `영업컨설팅파트 {YYMM} - {NN}` / 우 `모집인 / 교육용`

### 탭 유도 (중요 — 안 하면 아무도 안 누름)
FP는 "누를 수 있다"는 걸 모른다. 세 겹으로 걸 것:
1. 카드마다 화살표(›) 버튼 — 흰 원형 + 그림자
2. 그 화살표에서 **파장(ripple)** 이 번짐. 카드마다 시작 시점을 어긋나게
3. 리스트 위에 **네이비로 꽉 채운** 배지 — `👆 클릭해서 세부내용 보기` (17px, 펄스)
   - ⚠️ 옅은 색 배지는 배경에 묻혀 안 보인다 (2607 실패 경험)

### 디자인 톤
- 애플 프로덕트 페이지 문법: 극단적 여백, 큰 타이포, 박스·테두리 최소
- **전 페이지 밝은 톤 통일.** 표지만 옅은 블루-라벤더 그라데이션 (다크→화이트 급전환 금지)
- 색은 무채색 + 한화 네이비(`#12305C`) 액센트. 카테고리별 은은한 컬러 코딩
- 시각적 주인공 = **숫자**(입원일수)와 **빛**(그라데이션 오브). 텍스트 자료엔 넣을 사진이 없다
- 폰트 Pretendard (CDN)
- 헤더는 **가운데 정렬**로 통일. 카드 내부만 좌우 배치

### 카피
- 제목은 "무엇이 바뀌는가"가 아니라 **"무엇이 좋아지는가"**
  - ✗ "세 가지가 바뀝니다" (정보 없음)
  - ✓ "여성질환 3종이 열립니다" / "시스템 거절에서 선별 인수로"

## 원문 충실도 — HARD RULE
- 숫자·조건·담보명은 **공문 원문에서만**. 추측 금지
- 담보 약칭은 `mybrain/knowledge/base/담보명_약칭사전.md` **반드시 확인** (2607에 유갑생을 "담보군"으로 오독한 사고 있었음)
- 원문에 없는 결과 표현("인수 가능" vs "인수 검토")을 임의로 단정하지 말 것
- 제작 후 **공문과 1:1 대조**하고, FP가 해야 할 행동(기재 의무 등)이 빠지지 않았는지 확인

## 카톡 썸네일(OG) — 새 카드마다 필수 (2026-07-24 지시)
새 카드를 만들면 **항상** OG 썸네일을 함께 만든다 (빼먹으면 카톡 미리보기가 빈 링크로 보임):
1. 표지와 같은 팔레트로 가로형 컴포지션 HTML 작성 (800×418, eyebrow 배지 + 큰 제목 2줄 + 스탯 알약 + 고스트 텍스트 — si-premium/si-point og.png 참고)
2. headless Chrome `--window-size=800,418 --force-device-scale-factor=2` 로 `og.png`(1600×836) 캡처 → 카드 폴더에 저장
3. `<head>`에 og:title / og:description / og:url / og:image(+width/height) 삽입
4. 이미 카톡에 뿌려진 링크는 카카오가 OG를 캐시함 — https://developers.kakao.com/tool/debugger/sharing 에서 캐시 초기화 안내

## 완성 후 자동 미리보기 (2026-07-24 지시)
카드를 **최종 완성하면** 묻지 말고 자동으로 브라우저에 띄운다:
`python3 -m http.server 8899` (이미 떠 있으면 재사용) → `open "http://localhost:8899/{YYMM}/{슬러그}/"`
중간 작업 상태에서는 열지 않는다 — 경로 백틱 표시만.

## 이미지 뽑기 (교육자료·PPT 삽입용)
Chrome headless는 **window-size를 무시하고 뷰포트를 500px로 잡는다** → 그냥 찍으면 우측이 잘린다.
반드시 `_shot.html` 같은 **iframe 래퍼(393px 고정)** 를 거칠 것.

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless=new --disable-gpu --hide-scrollbars \
  --force-device-scale-factor=2 --window-size=393,852 --virtual-time-budget=5000 \
  --screenshot=out.png "http://localhost:8899/_shot.html?s=2&still=1"
```
→ 786×1704 (2x). 페이지는 `?s=N`(슬라이드) `?sheet=키`(시트 열기) `?still=1`(모션 끔) 지원.
