# 웹카드 사용측정 (self-hosted, 구글시트)

정적 호스팅(GitHub Pages)이라 서버 로그가 없다. 각 페이지에 경량 비컨(`track.js`)을
심어 이벤트를 **회사 구글계정의 Apps Script → 구글시트**로 보낸다. 외부 서비스 가입·쿠키
동의 배너 없음. 개인정보는 수집하지 않고 브라우저마다 랜덤 id만 쓴다.

## 무엇을 잡나
- 방문자 수(대략)·세션·재방문·모바일/PC
- 슬라이드 도달 깊이 / 이탈 지점 / 완주율
- 탭 참여율 + 항목별 탭 랭킹 (si-uw: 질환 항목·용어 / birth-uw: 탭·플랜·연령·체크)
- 세션 체류시간(평균)
- 자료별·날짜별 비교

## 못 잡는 것
- **유입 경로**: 카톡·밴드 인앱 브라우저는 referrer를 안 넘긴다 → 대부분 direct.
  채널 구분이 필요하면 배포 링크에 꼬리표(`?src=kakaoA`)를 손수 다르게 붙여 뿌린다.

## 설치 (한 번만)
1. 구글드라이브 → 새 **Google 스프레드시트** 생성 (이름 예: `webcard-analytics`)
2. 상단 **확장 프로그램 → Apps Script**
3. 기본 `Code.gs` 내용을 지우고 이 폴더의 **`Code.gs` 전체**를 붙여넣고 💾 저장
4. 우상단 **배포 → 새 배포**
   - 유형(톱니바퀴) → **웹 앱**
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자**
   - **배포** → 처음이면 권한 승인(내 계정 선택 → 고급 → 이동 → 허용)
5. 나온 **웹 앱 URL** 복사 (`https://script.google.com/macros/s/..../exec`)
6. `track.js`의 `ENDPOINT = "__PASTE_APPS_SCRIPT_URL__"` 를 그 URL로 교체 → 커밋·푸시
7. 끝. 같은 웹 앱 URL을 브라우저로 열면 **대시보드**가 보인다(새로고침 = 최신).

## 코드를 고쳤을 때
Apps Script는 **배포 → 배포 관리 → (연필) → 버전:새 버전 → 배포** 를 눌러야 반영된다.
(URL은 그대로 유지됨)

## 새 웹카드 만들 때
`</body>` 직전에 아래 한 줄만 넣으면 방문·슬라이드·체류가 자동 수집된다.
탭 신호는 클릭 핸들러에 `if(window.wc)wc('tap','키')` 한 줄씩 추가.
```html
<script src="../../analytics/track.js"></script>
```
